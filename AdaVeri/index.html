An interactive simulation framework for analysing LLM-assisted formal verification pipelines under stochastic success probabilities, global resets, and adaptive calibration.

§4 — Mathematical Foundation
§4.1–4.2 · Absorbing Markov Chains
Canonical Decomposition & Fundamental Matrix
The verification pipeline is modelled as a 6-state absorbing Markov chain. Transient states s₀–s₄ represent pipeline stages; s₅ (Verified) is the single absorbing state. The transition matrix admits the canonical block form separating transient dynamics Q from absorption routes R.
P = [Q R; 0 Iᵣ]  ·  N = (I − Q)⁻¹
§4.1.3–4.1.4 · Absorption Time & Tail Decay
Expected Steps & Spectral Radius
The fundamental matrix N gives expected visits per state. Absorption time follows a discrete phase-type distribution PH_d(π, Q). The spectral radius ρ(Q) governs the geometric rate of tail decay, enabling principled timeout configuration for safety-critical pipelines.
𝔼[τ | X₀=s₀] = e₀ᵀ N 1  ·  ℙ(τ>k) ≤ C · ρ(Q)ᵏ
§4.2 · Stochastic Regimes
Per-Step i.i.d. vs Per-Run Fixed
Two complementary regimes capture different variability sources. Per-step i.i.d.: each LLM call independently samples (Δ,Θ,Γ) — average matrix governs tail. Per-run fixed: task difficulty is persistent — tail is a mixture over ρ(Q) with worst-case bound λ_max.
i.i.d.: ρ(𝔼[Q])ᵏ  ·  per-run: λ_max = ess sup ρ(Q)
§4.3.1 · Analytical Derivation
Expected Absorption Time
By first-step analysis, the stochastic model yields 𝔼[τ] via the expected fundamental matrix. With no local retries, a closed form holds: the numerator sums per-stage sojourn costs 𝔼[1/Δᵢ]; the denominator penalises resets. Convergence requires Σ 𝔼[Γᵢ/Δᵢ] < 1.
v₁ = Σ𝔼[1/Δᵢ] / (1 − Σ𝔼[Γᵢ/Δᵢ])  →  4/δ when Γ=0
§4.4.1 · Online Estimation
Sliding-Window Estimator
During operation, empirical success rates are tracked over a rolling window W. Laplace smoothing prevents zero-probability estimates. The window balances responsiveness to drift vs. estimator stability. EWMA and CUSUM provide complementary drift-detection mechanisms.
Δ̂ᵢ = (sᵢ + 1) / (nᵢ + 2)  ·  trigger: Δ̂ᵢ < τ = 0.30
§4.4.2 · Bayesian Inference
Beta–Dirichlet Conjugate Posteriors
For per-run variability, a Dirichlet prior on (Δᵢ,Θᵢ,Γᵢ) yields closed-form posterior updates. Intervention fires on the posterior exceedance probability P(Δᵢ < τ | data) rather than a point estimate — eliminating false alarms when the window is sparse.
(Δ,Θ,Γ)|data ~ Dir(α+s, β+ℓ, γ+r)  ·  ε-threshold

Adaptive Calibration & Intervention
Dynamic monitoring — sliding-window estimator Δ̂ᵢ updated after every transition. Window W=20 by default, stratified per stage.
Intervention trigger — fires when Δ̂ᵢ < τ=0.30 (frequentist) or P(Δᵢ<τ|data) > ε=0.75 (Bayesian) with min 5 observations.
Stage-specific actions — s₁: context reset / prompt reformulation / temperature↑; s₂: syntax prompt / few-shot examples; s₃: invariant seed injection; s₄: solver fallback Z3→CVC5.
Resolution — intervention closes when Δ̂ᵢ ≥ 0.30 (or P(Δᵢ<τ|data) ≤ 0.40) with 5+ new observations.
Counterfactual gain — AdaVeri Report quantifies saved steps by comparing observed E[T] against E[T] recomputed with degraded Δ̂ values at trigger time, assuming no corrective action.
§6 — Dynamic Calibration Approaches
Frequentist · §6.1.1
Sliding-Window Monitoring
Tracks the most recent W transitions per stage in a rolling buffer. Point estimate Δ̂ᵢ = sᵢ/nᵢ is compared directly to threshold τ. Fast to compute, reacts immediately to drift, but sensitive to noise when the window is sparse. Intervention is deterministic once Δ̂ᵢ drops below τ. Recommended for stateless API calls with i.i.d. step variability.
Δ̂ᵢ(t) = (sᵢ + 1) / (nᵢ + 2)
trigger: Δ̂ᵢ < 0.30 and nᵢ ≥ 5
Bayesian · §6.1.3
Beta Conjugate Posterior
Maintains a Beta(α,β) posterior over each Δᵢ, updated on every transition. Prior is informed from configured nominal with strength κ. Intervention fires when P(Δᵢ < τ | data) > ε — the full posterior probability mass below threshold — eliminating false alarms when n is small. CI bands asymptotically narrow as 1/√n, showing theoretical convergence. Suited for persistent task-difficulty (per-run) regimes.
Δᵢ|data ~ Beta(α₀+k, β₀+n−k)
trigger: P(Δᵢ<τ | data) > ε
