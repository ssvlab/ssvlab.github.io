The Partnership

S3 Group International Partnership Programme

An open partnership framework between the Systems and Software Security (S3) Research Group at the University of Manchester and organisations worldwide — industry, research institutes, and technology companies.

Covers cooperation in Formal Verification and Software Security, with a focus on AI-assisted systems, embedded systems, and safety-critical applications. Two-Way Exchange — Visiting Researcher Programme

Researchers and engineers nominated by partner organisations can come to Manchester as Visiting Researchers (PGR). They return with cutting-edge knowledge, tools, and a direct connection to the global innovation ecosystem. The programme is fully structured and active.

Tools, Publications and Research Frontiers

Award-Winning Tools

ESBMC

Model checker for C/C++/Python/Java. 50+ awards at SV-COMP and Test-Comp (2012–2025).

esbmc.org →

JBMC

Formal verification of Java programs using model checking.

FuSeBMC

Code coverage combining fuzzing, BMC, and static analysis.

Vampire / iProver

High-performance theorem provers for automated reasoning.

Recent Publications (2024–25)

  • LLMs + model checking for automatic invariant generation
  • ESBMC-Python: formal verifier for Python
  • SecureFalcon: vulnerability detection with LLMs (IEEE TSE)
  • Self-healing software with LLMs + formal verification
  • Verification of Arm Confidential Computing Architecture components
  • CASTLE: benchmark for static analysis and LLMs in CWE detection

Open Challenges in Formal Methods

Scalability

Verifying programs with millions of lines remains computationally expensive.

Concurrent Software

Race conditions and deadlocks in multithreaded and distributed systems.

AI/ML Verification

Guaranteeing formal properties in neural networks and autonomous systems.

LLM-Generated Code

Assessing and guaranteeing the safety of code generated by language models.

Modern Hardware

Verification on capability architectures (Arm Morello/CHERI).

Cyber-Physical Systems

Embedded digital controllers with real-time requirements.

Steps to Formalise the Partnership

📄 Partnership Guide — governing document of this collaboration

The Partnership Guide explains in detail how this partnership works: roles, contribution types, IP framework, data protection, commercialisation, and more. Read it before your first meeting.

Download Partnership Guide (PDF) →
✉️

1. Support Letter

The organisation signs a letter endorsing a project in submission. No financial cost — high technical and strategic value for both sides.

No financial cost High visibility
🔍

2. Technical Use Case

Map a real problem from your organisation to address jointly — embedded software verification, IoT security, critical systems compliance, and more.

Applied R&D S3 Tools & Expertise
👤

3. Technical Point of Contact

Nominate a researcher or engineer from your organisation to attend technical meetings, follow projects, and act as the link with the S3 Group at UoM.

Periodic meetings Direct access to S3
🎓

4. Visiting Researcher (optional)

Nominate a researcher from your organisation for an immersion period in Manchester as a Visiting PGR. Requirements: English (IELTS ≥ 6.5 or TOEFL iBT ≥ 79) and a proposal aligned with the S3 Group.

Immersion in Manchester Access to UoM labs
📄

5. Formal Project Partner

In the next relevant grant call, the organisation is formally included as a project partner with a technical (in-kind) contribution — opening access to UK and international funding.

EPSRC Royal Society Innovate UK
🚀

How the S3 Group Supports the Partnership

Pierre Vilar Dantas, Postdoctoral Researcher and Commercial Lead at UoM, is the technical point of contact for partner organisations — working across three simultaneous fronts.

Track 1 — Onboarding & Mobility

  • Co-authorship and review of scientific papers
